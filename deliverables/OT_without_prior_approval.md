Overtime Without Prior Approval — Complete Flow
                                                                                                                                                                                                  Step 1: Shift End Detection

  - System detects shift is ending 30 minutes before scheduled end
  - Checks if employee has pre-approved OT → if yes, no popup
  - If no pre-approved OT → sends notification via socket

  Step 2: Employee Gets Popup

  - "Shift Ending" modal appears with warning:
  ▎ "Your shift ends at [time]. If you continue working, the extra time will be tracked as overtime without prior approval. You may not be compensated for unapproved overtime."
  - Employee must provide a reason to continue
  - Two options: "No, I'm Good" or "Continue Working"

  Step 3: Controlled Pause (at exact shift end time)

  - If employee hasn't responded, system initiates a 2-minute controlled pause
  - "Shift Has Ended" modal with countdown
  - Options: "Stay Clocked Out" or "Continue Working"
  - If no response after 2 minutes → auto-clocked out at scheduled shift end time

  Step 4: Employee Clocks Out

  When the employee finally clocks out, the system:
  1. Calculates shiftExtensionMinutes (time worked past scheduled end, with 7-min grace period)
  2. Creates/updates TimeRecord with:
    - shiftExtensionMinutes = minutes past shift end
    - shiftExtensionStatus = UNAPPROVED
    - shiftExtensionReason = employee's reason from modal
    - status = PENDING
  3. Auto-creates an OvertimeRequest record:
    - Type: SHIFT_EXTENSION
    - Status: PENDING
    - isAutoGenerated: true
    - Reason: "Auto-generated — employee worked overtime without prior request"
  4. Notifies the client via in-app notification, email, and SMS

  Step 5: Client Approves or Rejects

  Client sees the pending OT in their portal and can:

  ┌─────────────┬─────────────────────────────────┬─────────────────────────────────────────────┐
  │   Action    │          What Happens           │               Payroll Impact                │
  ├─────────────┼─────────────────────────────────┼─────────────────────────────────────────────┤
  │ Approve     │ shiftExtensionStatus → APPROVED │ OT hours paid at OT rate                    │
  ├─────────────┼─────────────────────────────────┼─────────────────────────────────────────────┤
  │ Reject      │ shiftExtensionStatus → DENIED   │ OT hours excluded, only regular hours paid  │
  ├─────────────┼─────────────────────────────────┼─────────────────────────────────────────────┤
  │ No response │ Stays PENDING/UNAPPROVED        │ OT hours NOT included in payroll or invoice │
  └─────────────┴─────────────────────────────────┴─────────────────────────────────────────────┘

  Step 6: Payroll

  - Only APPROVED OT is included:
    - regularPay = regularHours × hourlyRate
    - overtimePay = approvedOTHours × overtimeRate
  - UNAPPROVED/DENIED OT is excluded — employee only gets regular hours pay
  - If client approves after payroll is finalized → system creates a PayrollAdjustment (bonus) in the next payroll period

  Step 7: Invoice

  - Same as payroll — only APPROVED OT is invoiced to the client
  - UNAPPROVED OT is excluded completely
  - Pre-invoice warning sent to client if pending OT exists: "Unapproved OT will NOT be included in this invoice"
